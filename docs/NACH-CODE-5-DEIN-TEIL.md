# Nach Code 5 – was du (Georg) machst

**Zweck:** Damit wir genau dort weiterarbeiten können, wo wir aufgehört haben. Du machst wenig, die KI übernimmt den Rest.

---

## Deine Schritte (jedes Mal nach Code 5)

1. **Fenster wiederherstellen**  
   „Reopen“ klicken (oder Cursor neu starten).

2. **Änderungen sichern (falls noch offen)**  
   **Cmd+S** oder **File → Save All**. So sind alle Änderungen auf der Platte – beim Reopen geht nichts verloren. Nicht „Don’t restore“ / Option wählen, die Editor-Inhalte verwirft.

3. **Einen kurzen Satz an die KI schreiben**  
   Z.B.:  
   - **„weiter nach Code 5“**  
   - **„weiter“**  
   - **„wo waren wir“**  

   Mehr brauchst du nicht.

---

## Was die KI dann macht (ohne dass du mehr tun musst)

- Liest **docs/DIALOG-STAND.md** (dort steht: woran wir zuletzt gearbeitet haben, **Nächster Schritt**).
- Liest bei Bedarf **docs/WEITERARBEITEN-NACH-ABSTURZ.md** und **docs/CRASH-BEREITS-GEPRUEFT.md**.
- Arbeitet am **Nächsten Schritt** weiter und fragt nicht „womit soll ich anfangen?“.

---

## Damit das funktioniert

Am Ende jeder Aufgabe schreibt die KI in **DIALOG-STAND.md** den **Nächsten Schritt** (konkret, z.B. „Build prüfen, dann Commit“ oder „GalerieVorschauPage: Filter für X anpassen“). So weiß sie nach Code 5 sofort, wo es weitergeht.

---

**Kurz:** Nach Code 5 → Reopen → ggf. Save All → „weiter“ sagen. Den Rest übernimmt die KI mit DIALOG-STAND.
