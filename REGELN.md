# Projekt-Regeln (K2 Galerie)

Regeln sind verbindlich – egal wo sie stehen. Man muss sich darauf verlassen können.

---

## Bei Unsicherheit: nachfragen

**Regel:** Wenn unsicher (ob etwas so gemeint ist, ob eine Regel greift, ob eine Ablage stimmt): **nicht raten, sondern den Nutzer fragen.** Das gilt immer.

---

## Stand nach Umsetzung

**Regel:** Sobald eine Umsetzung (Feature, Fix, Änderung) erledigt ist, **sofort** einen neuen Stand setzen.

- **Wie:** `node scripts/write-build-info.js` ausführen (oder `npm run dev` / `npm run build` – die rufen das Script auf).
- **Warum:** Keine abgeschlossene Änderung ohne aktualisierten Stand; sonst ist nicht erkennbar, ob etwas schon drin ist.

Details: **STAND-VERSION-GERAETE.md**.
