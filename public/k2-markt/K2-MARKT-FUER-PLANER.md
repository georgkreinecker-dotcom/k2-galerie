# K2 Markt – Für die Planer

**Stand:** 09.03.26  
**Zweck:** Architektur steht. Hier steigen die Planer ein: Roadmap, Phasen, Meilensteine, Reihenfolge – alles auf Basis der **einen** Vision und Architektur.

---

## 0. Wer plant (Ressourcen)

**Es gibt keine externen Planer oder ein separates Team.** Planung läuft mit dem, was da ist: **du + KI.** **Ziel: So wenig wie möglich bleibt bei dir** – die KI übernimmt alles Machbare (Strukturieren, Phasen vorschlagen, Ausarbeiten, Festhalten). Du gibst höchstens die Richtung vor; den Rest erledigt die KI. Du in Pension – Entlastung, nicht Zusatzlast. Keine Erwartung an „Ressourcen, auf die man zugreift“.

## 2. Was die Vision als nächste Schritte nennt (Eingabe für die Planung)

| Nr. | Inhalt (aus Vision §10) |
|-----|------------------------|
| 1 | Produkt-Moment modellieren (Felder: Botschaft, Zielgruppe, Kontakt, Links, Medien …) |
| 2 | Definition of Done pro Format (Flyer, E-Mail, Presse, …) |
| 3 | Ersten Agenten als Ableiter (z. B. Flyer-Agent: Input = Produkt-Moment + Template; Output = Entwurf) |
| 4 | Tor + Freigabe in der UI (Bildschirm, Prüfung, Button „Freigeben“) |
| 5 | Traceability von Anfang an (Referenz auf Moment, Template, Regel-Version) |

---

## 3. Der Fluss (zum Einordnen)

```
Produkt-Moment (eine Wahrheit) → Agenten (Entwurf pro Format) → Qualitäts-Tor (DoD?) → Freigabe → Markt
```

---

## 4. Phasen und Meilensteine (Stand 09.03.26 – Start)

| Phase | Inhalt | Meilenstein „fertig“ |
|-------|--------|------------------------|
| **1** | Produkt-Moment modellieren + Definition of Done (Mindest-Anzahl Formate) | Struktur Produkt-Moment definiert; DoD für mind. 1 Format (z. B. Flyer) festgelegt. |
| **2** | Erster Agent als Ableiter (z. B. Flyer-Agent) | Ein Agent liest Produkt-Moment + ein Template, liefert Entwurf. |
| **3** | Tor + Freigabe in der UI | Bildschirm zeigt Entwurf, Prüfung (DoD?), Button „Freigeben“ – erst dann marktfähig. |
| **4** | Traceability + weitere Formate | Jedes Output referenziert Moment, Template, Regel-Version; weitere Formate nach gleichem Muster. |

**Reihenfolge:** 1 → 2 → 3 → 4 (ohne Produkt-Moment kein Agent, ohne Agent kein Tor, ohne Tor keine Freigabe).

**Start:** Phase 1 – siehe [Produkt-Moment (Modell)](K2-MARKT-PRODUKT-MOMENT.md).
