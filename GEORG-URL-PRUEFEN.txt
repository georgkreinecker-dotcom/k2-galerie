GEORG:

In .env steht:
→ VITE_SUPABASE_URL=https://sjayeqnibwyxtwzcqklj.supabase.co

Aber die URL könnte falsch sein!

═══════════════════════════════════════════════════════════════

LÖSUNG: URL prüfen

Schritt 1: Richtige URL kopieren
→ Supabase Dashboard → Settings → API
→ Project URL: Kopiere diese URL
→ Sollte sein: https://sjqyeqnibwyxtwzcqklj.supabase.co
  (mit "q" nicht "y")

Schritt 2: .env korrigieren
→ Öffne .env Datei
→ Prüfe VITE_SUPABASE_URL
→ Vergleiche mit Supabase Dashboard
→ Falls unterschiedlich: Korrigiere
→ Speichern: Cmd+S

Schritt 3: App neu starten
→ Terminal: Stoppe App (Ctrl+C)
→ Neu starten: npm run dev
→ Warten bis App läuft

═══════════════════════════════════════════════════════════════

WICHTIG:

Die URL muss GENAU übereinstimmen!
→ Jeder Buchstabe muss gleich sein
→ Jeder Punkt muss gleich sein
→ .co nicht .com!

═══════════════════════════════════════════════════════════════

HAUFIGE FEHLER:

Fehler 1: Tippfehler
→ sjayeqnibwyxtwzcqklj (mit "y")
→ sjqyeqnibwyxtwzcqklj (mit "q")
→ Unterschied: "y" vs "q"!

Fehler 2: Falsche Endung
→ .supabase.com (falsch)
→ .supabase.co (richtig)

═══════════════════════════════════════════════════════════════

BITTE PRÜFE:

1. Supabase Dashboard → Settings → API → Project URL
2. Kopiere die URL
3. Vergleiche mit .env Datei
4. Sind sie identisch? (Ja/Nein)

Falls NEIN:
→ Korrigiere in .env
→ App neu starten
→ Nochmal testen
